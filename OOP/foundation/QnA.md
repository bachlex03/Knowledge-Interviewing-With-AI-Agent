# OOP Foundation Q&A

### Level 1: Remembering

#### Q_LEVEL1_101: What is Object-Oriented Programming (OOP)?

**Question:**
en: What is Object-Oriented Programming (OOP)?
vi: Lập trình hướng đối tượng (OOP) là gì?

**Answer:**
en: OOP is a programming paradigm based on the concept of "objects", which can contain data (attributes) and code (methods).
vi: OOP là một mô hình lập trình dựa trên khái niệm về các "đối tượng", có thể chứa dữ liệu (thuộc tính) và mã (phương thức).

#### Q_LEVEL1_102: What are the four main principles of OOP?

**Question:**
en: What are the four main principles of OOP?
vi: Bốn nguyên lý chính của OOP là gì?

**Answer:**
en: The four main principles are Encapsulation, Abstraction, Inheritance, and Polymorphism.
vi: Bốn nguyên lý chính là Đóng gói (Encapsulation), Trừu tượng (Abstraction), Kế thừa (Inheritance), và Đa hình (Polymorphism).

#### Q_LEVEL1_103: What is a Class?

**Question:**
en: What is a Class in OOP?
vi: Lớp (Class) trong OOP là gì?

en: A class is a blueprint or template from which objects are created. It defines properties and behaviors.
vi: Một lớp là một bản thiết kế hoặc khuôn mẫu để từ đó tạo ra các đối tượng. Nó định nghĩa các thuộc tính và hành vi.

#### Q_LEVEL1_104: What is an Object?

**Question:**
en: What is an Object?
vi: Đối tượng (Object) là gì?

**Answer:**
en: An object is an instance of a class containing specific data and behaviors.
vi: Một đối tượng là một thực thể (instance) của một lớp, chứa dữ liệu và hành vi cụ thể.

#### Q_LEVEL1_105: What is Encapsulation?

**Question:**
en: What is Encapsulation?
vi: Tính đóng gói (Encapsulation) là gì?

**Answer:**
en: Encapsulation is the bundling of data and methods that operate on that data within a single unit, restricting direct access to some components.
vi: Tính đóng gói là việc gom nhóm dữ liệu và các phương thức hoạt động trên dữ liệu đó vào một đơn vị duy nhất, hạn chế quyền truy cập trực tiếp vào một số thành phần.

#### Q_LEVEL1_106: What is Abstraction?

**Question:**
en: What is Abstraction?
vi: Tính trừu tượng (Abstraction) là gì?

**Answer:**
en: Abstraction is hiding complex implementation details and showing only the essential features of an object.
vi: Tính trừu tượng là việc ẩn đi các chi tiết triển khai phức tạp và chỉ hiển thị những tính năng thiết yếu của một đối tượng.

#### Q_LEVEL1_107: What is Inheritance?

vi: Tính kế thừa (Inheritance) là gì?

vi: Cung cấp một triển khai mới trong lớp dẫn xuất (derived class) cho một phương thức đã được định nghĩa ở lớp cơ sở (base class).

en: What is the 'base' keyword in C#?

### Level 2: Understanding

#### Q_LEVEL2_201: Explain the difference between a Class and an Object.

**Question:**
en: Explain the difference between a Class and an Object.
vi: Giải thích sự khác biệt giữa Class và Object.

**Answer:**
en: A Class is a logical blueprint, whereas an Object is a physical reality that occupies memory based on that blueprint.
vi: Class (Lớp) là một bản thiết kế logic, trong khi Object (Đối tượng) là một thực thể vật lý chiếm bộ nhớ được tạo ra dựa trên bản thiết kế đó.

#### Q_LEVEL2_202: How does Abstraction differ from Encapsulation?

**Question:**
en: How does Abstraction differ from Encapsulation?
vi: Tính trừu tượng khác với Tính đóng gói như thế nào?

**Answer:**
en: Abstraction hides complexity by exposing only necessary features. Encapsulation hides internal state and restricts direct access to it.
vi: **Vấn đề & Giải pháp:** Tính trừu tượng tập trung vào việc ẩn đi sự phức tạp bằng cách chỉ phơi bày các tính năng cần thiết (WHAT). Tính đóng gói tập trung vào việc bảo vệ dữ liệu nội bộ và ngăn chặn truy cập trực tiếp từ bên ngoài (HOW).

#### Q_LEVEL2_203: Why is Inheritance useful?

**Question:**
en: Why is Inheritance useful in software development?
vi: Tại sao tính kế thừa lại hữu ích trong phát triển phần mềm?

**Answer:**
en: It promotes code reusability and establishes a logical "IS-A" relationship, reducing redundancy.
vi: **Vấn đề & Giải pháp:** Thay vì viết lại code giống nhau cho các đối tượng có chung đặc điểm, Kế thừa giúp tái sử dụng mã nguồn, thiết lập mối quan hệ "IS-A" (Là một) logic và giúp dễ bảo trì hơn.

#### Q_LEVEL2_204: Explain the benefit of Polymorphism.

**Question:**
en: Explain the primary benefit of Polymorphism.
vi: Giải thích lợi ích chính của Tính đa hình.

**Answer:**
en: It allows systems to treat objects of different types uniformly, making code more flexible and scalable.
vi: **Vấn đề & Giải pháp:** Nó cho phép hệ thống xử lý các đối tượng thuộc các loại khác nhau một cách thống nhất thông qua chung một interface. Nhờ vậy, code trở nên linh hoạt, dễ dàng mở rộng khi thêm các đối tượng mới mà không cần sửa đổi code cũ.

#### Q_LEVEL2_205: Compare Method Overloading vs Overriding.

**Question:**
en: Compare Method Overloading and Method Overriding.
vi: So sánh Nạp chồng phương thức (Overloading) và Ghi đè phương thức (Overriding).

**Answer:**
en: Overloading happens at compile-time within the same class (different parameters). Overriding happens at run-time between base and derived classes (same parameters).
vi: **Overloading** xảy ra lúc biên dịch (compile-time) trong cùng một lớp với các tham số khác nhau. **Overriding** xảy ra lúc chạy (run-time) giữa lớp cơ sở và lớp dẫn xuất, yêu cầu tham số hoàn toàn giống nhau.

#### Q_LEVEL2_206: Interface vs Abstract Class.

**Question:**
en: What is the difference between an Interface and an Abstract Class?
vi: Sự khác biệt giữa Interface và Abstract Class là gì?

**Answer:**
en: An Abstract Class can have implementations and state, while an Interface typically only defines a contract (though modern C# allows default implementations). A class can implement multiple interfaces but inherit from only one abstract class.
vi: **Vấn đề:** Khi thiết kế cấu trúc lớp, chúng ta cần cân nhắc. Abstract Class có thể chứa mã đã triển khai và các trường dữ liệu (trạng thái), một lớp chỉ có thể kế thừa một Abstract Class. Interface thường chỉ chứa khai báo (hợp đồng) và một lớp có thể triển khai nhiều Interface.

en: Why do we need Constructors?
vi: Tại sao chúng ta cần Hàm tạo (Constructors)?

#### Q_LEVEL2_208: Explain the 'virtual' keyword in C#.

#### Q_LEVEL2_209: Explain the 'sealed' keyword.

**Question:**
en: Explain the 'sealed' keyword in C#.
vi: Giải thích từ khóa 'sealed' trong C#.

vi: Nó được dùng để ngăn chặn một lớp không cho phép lớp khác kế thừa, hoặc ngăn chặn một phương thức đã bị ghi đè (overridden) không cho phép ghi đè thêm ở các lớp con thấp hơn nữa.

#### Q_LEVEL2_210: Public vs Private vs Protected.

**Question:**
en: Describe the difference between public, private, and protected.
vi: Mô tả sự khác biệt giữa public, private và protected.

**Answer:**
en: Public is accessible anywhere; Private is accessible only within the same class; Protected is accessible within the class and its derived classes.
vi: **Public**: Có thể truy cập từ bất cứ đâu. **Private**: Chỉ có thể truy cập bên trong chính lớp đó. **Protected**: Có thể truy cập bên trong lớp đó và các lớp dẫn xuất (kế thừa) từ nó.

#### Q_LEVEL2_211: Encapsulation and Security.

**Question:**
en: How does encapsulation improve software security?
vi: Tính đóng gói cải thiện độ an toàn của phần mềm như thế nào?

en: It prevents accidental or malicious manipulation of internal state by forcing interactions through well-defined public methods.

#### Q_LEVEL2_212: "IS-A" vs "HAS-A".

**Question:**
en: Explain the concept of "IS-A" vs "HAS-A" relationship.
vi: Giải thích khái niệm về mối quan hệ "IS-A" và "HAS-A".

**Answer:**
en: "IS-A" implies Inheritance (a Dog is an Animal). "HAS-A" implies Composition (a Car has an Engine).
vi: **IS-A** (Là một) thể hiện tính kế thừa (ví dụ: Chó là một Động vật). **HAS-A** (Có một) thể hiện tính bao gộp / thành phần (Composition), một đối tượng chứa đối tượng khác (ví dụ: Ô tô có một Động cơ).

#### Q_LEVEL2_213: Early vs Late Binding.

**Question:**
en: What is early binding vs late binding?
vi: Early binding (Liên kết sớm) và Late binding (Liên kết trễ) là gì?

**Answer:**
en: Early binding is resolved at compile-time (e.g., method overloading). Late binding is resolved at run-time (e.g., method overriding/virtual methods).
vi: **Early binding** được trình biên dịch quyết định ngay lúc biên dịch (ví dụ: Nạp chồng phương thức). **Late binding** được quyết định lúc chương trình đang chạy dựa trên kiểu thực tế của đối tượng (ví dụ: Đa hình / Ghi đè phương thức).

#### Q_LEVEL2_215: Properties vs Fields.

**Question:**

vi: Properties (Thuộc tính) khác với Fields (Trường) trong C# như thế nào?

### Level 3: Applying

#### Q_LEVEL3_301: Demonstrate Encapsulation.

**Question:**
en: Demonstrate Encapsulation using a C# class.
vi: Minh họa Tính đóng gói (Encapsulation) bằng một lớp C#.

**Answer:**
en: Use private fields and public properties to control data access.
vi: Sử dụng các trường private và các thuộc tính public để kiểm soát truy cập dữ liệu.

```csharp
public class BankAccount
{
    // Private field - encapsulated data
    private decimal balance;

    // Public property to control access
    public decimal Balance
    {
        get { return balance; }
        private set { balance = value; } // Only the class can change it directly
    }

    public void Deposit(decimal amount)
    {
        if (amount > 0)
        {
            balance += amount; // Validated modification
        }
    }
}
```

#### Q_LEVEL3_302: Implement Inheritance.

**Question:**
en: Implement Inheritance between a base class and a derived class in C#.
vi: Triển khai Tính kế thừa giữa một lớp cơ sở và một lớp dẫn xuất trong C#.

**Answer:**
en: Create an Animal base class and a Dog derived class.
vi: Tạo một lớp cơ sở Animal và một lớp dẫn xuất Dog kế thừa từ nó.

```csharp
// Base class
public class Animal
{
    public void Eat()
    {
        Console.WriteLine("Eating...");
    }
}

// Derived class inherits from Animal

{
    public void Bark()
    {
        Console.WriteLine("Barking...");
    }
}

// Usage:
// var myDog = new Dog();
// myDog.Eat();  // Inherited method
// myDog.Bark(); // Dog's own method
```

#### Q_LEVEL3_303: Compile-time Polymorphism (Overloading).

**Question:**
en: Show an example of Compile-time Polymorphism (Overloading) in C#.
vi: Chỉ ra một ví dụ về Tính đa hình lúc biên dịch (Nạp chồng) trong C#.

**Answer:**
en: Create a MathHelper class with overloaded Add methods.
vi: Tạo một lớp MathHelper với các phương thức Add được nạp chồng.

```csharp
public class MathHelper
{
    // Overload 1: Two integers
    public int Add(int a, int b)
    {
        return a + b;
    }

    // Overload 2: Three integers
    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    // Overload 3: Two doubles
    public double Add(double a, double b)
    {
        return a + b;
    }
}
```

#### Q_LEVEL3_304: Run-time Polymorphism (Overriding).

**Question:**
en: Show an example of Run-time Polymorphism (Overriding) in C#.
vi: Chỉ ra một ví dụ về Tính đa hình lúc chạy (Ghi đè) trong C#.

**Answer:**
en: Use virtual and override keywords.
vi: Sử dụng các từ khóa virtual và override.

```csharp
public class Shape
{
    // Virtual method can be overridden
    public virtual void Draw()
    {
        Console.WriteLine("Drawing a generic shape");
    }
}

public class Circle : Shape
{
    // Override provides specific implementation
    public override void Draw()
    {
        Console.WriteLine("Drawing a Circle");
    }
}

// Usage:
// Shape myShape = new Circle();
// myShape.Draw(); // Outputs: "Drawing a Circle" (Late binding)
```

#### Q_LEVEL3_305: Implementing an Interface.

**Question:**
en: Demonstrate how to implement an Interface in a C# class.
vi: Minh họa cách triển khai một Interface trong một lớp C#.

**Answer:**
en: Define an ILogger interface and implement it in a FileLogger class.
vi: Định nghĩa một interface ILogger và triển khai nó trong lớp FileLogger.

```csharp
// Define the interface contract
public interface ILogger
{
    void Log(string message);
}

// Implement the interface
public class FileLogger : ILogger
{
    public void Log(string message)
    {
        // Concrete implementation for logging to a file
        Console.WriteLine($"Writing to file: {message}");
    }
}
```
