#### Q_LEVEL3_601: Apply a simple story prioritization model

**Question:**
en: How would you model a simple Agile backlog prioritization rule in C#?
vi: Bạn sẽ mô hình hóa một quy tắc ưu tiên backlog đơn giản trong C# như thế nào?

**Answer:**
en: One practical approach is to score backlog items by business value, urgency, and effort, then sort by the best value-to-cost ratio. The goal is not mathematical perfection but making prioritization explicit and discussable.
vi: Một cách thực tế là chấm điểm backlog item dựa trên business value, urgency và effort, rồi sắp xếp theo tỷ lệ giá trị trên chi phí. Mục tiêu không phải là toán học hoàn hảo mà là làm cho việc ưu tiên trở nên minh bạch và dễ thảo luận.

```csharp
public sealed record BacklogItem(string Title, int BusinessValue, int Urgency, int Effort);

var backlog = new List<BacklogItem>
{
    new("Improve checkout error message", 8, 9, 2),
    new("Add dark mode", 4, 2, 5),
    new("Fix payment retry bug", 10, 10, 3)
};

var ordered = backlog
    .OrderByDescending(item => (item.BusinessValue + item.Urgency) / (double)item.Effort)
    .ToList();

foreach (var item in ordered)
{
    Console.WriteLine(item.Title);
}
```

#### Q_LEVEL3_614: Apply a Definition of Done checklist

#### Q_LEVEL3_639: Apply a sprint capacity calculation

**Question:**
en: How would you estimate sprint capacity with a simple C# model?
vi: Bạn sẽ ước tính sprint capacity bằng một mô hình C# đơn giản như thế nào?

**Answer:**
en: Start with team availability, subtract time lost to planned leave or fixed overhead, and use the result as a capacity guide rather than a hard promise.
vi: Hãy bắt đầu từ tổng thời gian sẵn sàng của team, trừ đi thời gian nghỉ đã biết hoặc overhead cố định, rồi dùng kết quả đó như một hướng dẫn capacity chứ không phải lời hứa cứng.

```csharp
var teamMembers = 5;
var sprintDays = 10;
var hoursPerDay = 6;
var plannedLeaveHours = 18;

var grossCapacity = teamMembers * sprintDays * hoursPerDay;
var netCapacity = grossCapacity - plannedLeaveHours;

Console.WriteLine($"Net sprint capacity: {netCapacity} hours");
```

#### Q_LEVEL3_651: Apply a retrospective action tracker

**Question:**
en: How would you track retrospective action items in a simple C# model?
vi: Bạn sẽ theo dõi action item của retrospective bằng một mô hình C# đơn giản như thế nào?

**Answer:**
en: Retrospectives only matter when improvement actions are visible and owned. A lightweight model can track who owns each action and whether it is completed.
vi: Retrospective chỉ có ý nghĩa khi action cải tiến được nhìn thấy rõ và có owner. Một mô hình nhẹ có thể theo dõi ai chịu trách nhiệm và action đó đã hoàn thành hay chưa.

