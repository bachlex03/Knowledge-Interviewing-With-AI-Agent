#### Q_LEVEL2_321: Why does Kanban use WIP limits?

**Question:**
en: Why are WIP limits central to Kanban?
vi: Vì sao WIP limit là trọng tâm của Kanban?

**Answer:**
en: WIP limits reduce multitasking, expose bottlenecks, and force the system to finish work before starting more. This improves flow and often shortens cycle time.
vi: WIP limit giúp giảm multitasking, làm lộ bottleneck và buộc hệ thống phải hoàn thành việc đang làm trước khi bắt đầu thêm việc mới. Nhờ đó flow tốt hơn và cycle time thường ngắn hơn.

#### Q_LEVEL2_334: Why is visualization important?

en: Why is visualization important in Kanban?
vi: Vì sao trực quan hóa lại quan trọng trong Kanban?

**Answer:**

vi: Trực quan hóa giúp biến phần công việc vô hình và độ trễ trở nên nhìn thấy được. Khi team thấy rõ queue, blocked item và trạng thái quá tải, việc cải tiến sẽ dễ hơn và khách quan hơn.

#### Q_LEVEL2_347: Pull vs push

**Question:**
en: How is a pull system different from a push system?
vi: Hệ pull khác hệ push như thế nào?

**Answer:**
en: In a push system, work is assigned regardless of current capacity. In a pull system, work enters only when the downstream stage has room to handle it.
vi: Trong hệ push, công việc được giao xuống bất kể capacity hiện tại. Trong hệ pull, công việc chỉ đi vào khi stage downstream còn chỗ để xử lý.

#### Q_LEVEL3_521: Apply a WIP limit checker

**Question:**
en: How would you model a simple WIP limit checker in C# for a Kanban column?
vi: Bạn sẽ mô hình hóa một bộ kiểm tra WIP limit đơn giản trong C# cho một cột Kanban như thế nào?

**Answer:**
en: Compare the number of active items in a column with its configured WIP limit, then raise a signal when the limit is exceeded.
vi: Hãy so sánh số item đang hoạt động trong một cột với WIP limit đã cấu hình, sau đó phát tín hiệu khi giới hạn bị vượt quá.

```csharp
public sealed record KanbanColumn(string Name, int WipLimit, int ActiveItems);

var column = new KanbanColumn("In Progress", WipLimit: 3, ActiveItems: 5);
var isOverLimit = column.ActiveItems > column.WipLimit;

Console.WriteLine($"{column.Name} over limit: {isOverLimit}");
```

#### Q_LEVEL3_534: Apply cycle time calculation

**Question:**
en: How would you calculate cycle time for a work item in C#?
vi: Bạn sẽ tính cycle time cho một work item trong C# như thế nào?

**Answer:**
en: Store the start and completion timestamps, then subtract them to get the time spent in progress.
vi: Hãy lưu thời điểm bắt đầu và thời điểm hoàn thành, rồi lấy chúng trừ cho nhau để ra thời gian item nằm trong trạng thái xử lý.

```csharp
var startedAt = new DateTime(2026, 4, 1, 9, 0, 0, DateTimeKind.Utc);
var finishedAt = new DateTime(2026, 4, 3, 15, 30, 0, DateTimeKind.Utc);

var cycleTime = finishedAt - startedAt;
Console.WriteLine($"Cycle time: {cycleTime.TotalHours} hours");
```

#### Q_LEVEL3_576: Apply replenishment selection

**Question:**
en: How would you choose which work item to pull next into a Kanban system?
vi: Bạn sẽ chọn work item nào để kéo vào hệ thống Kanban tiếp theo như thế nào?

**Answer:**
en: Use an explicit pull rule based on class of service, readiness, and available capacity so the choice is transparent and consistent.
vi: Hãy dùng quy tắc pull rõ ràng dựa trên class of service, mức độ sẵn sàng và capacity còn lại để việc lựa chọn minh bạch và nhất quán.

```csharp
public sealed record Candidate(string Title, int PriorityScore, bool Ready);

var nextItem = new List<Candidate>
{
    new("Fix checkout timeout", 10, true),
    new("Refactor old reporting module", 4, true),
    new("Investigate dashboard redesign", 6, false)
}
.Where(x => x.Ready)
.OrderByDescending(x => x.PriorityScore)
.First();

Console.WriteLine($"Next pull item: {nextItem.Title}");
```
